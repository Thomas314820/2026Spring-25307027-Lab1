# 贡献指南

感谢您考虑为贪吃蛇游戏项目做出贡献！

## 🤝 如何贡献

### 报告 Bug

如果您发现了 bug，请创建一个 Issue 并包含以下信息：

- 清晰的标题和描述
- 复现步骤
- 预期行为和实际行为
- 截图（如果适用）
- 设备和环境信息

### 建议新功能

如果您有新功能的想法，请创建一个 Issue 并标记为 `enhancement`：

- 清晰描述该功能
- 解释为什么这个功能有用
- 提供可能的实现方案

### 提交代码

1. **Fork 仓库**
   ```bash
   git clone https://github.com/yourusername/harmonyos-snake-game.git
   ```

2. **创建分支**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **进行更改**
   - 遵循代码规范
   - 添加必要的注释
   - 保持代码简洁清晰

4. **提交更改**
   ```bash
   git add .
   git commit -m "Add: your feature description"
   ```

5. **推送到分支**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **创建 Pull Request**
   - 提供清晰的 PR 标题和描述
   - 关联相关的 Issue
   - 等待代码审查

## 📝 代码规范

### ArkTS/TypeScript 规范

- 使用 2 空格缩进
- 使用有意义的变量和函数名
- 添加必要的注释
- 遵循 ArkTS 语法规范

### 提交信息规范

使用清晰、描述性的提交信息：

- `Add:` 添加新功能
- `Fix:` 修复 bug
- `Update:` 更新现有功能
- `Refactor:` 重构代码
- `Docs:` 文档更新
- `Style:` 代码格式调整

提交 PR 前，请确保：

- [ ] 代码遵循项目规范

- [ ] 更新了相关文档

如果您有任何问题，可以：
