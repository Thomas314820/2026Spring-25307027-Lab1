## 游戏流程

### 1. 开始游戏
```
应用启动 → 初始化游戏 → 蛇出现在左侧 → 食物随机生成 → 开始移动
```

### 2. 游戏进行
```
定时器触发 → 移动蛇身 → 检测碰撞 → 更新状态 → 刷新UI
    ↓
吃到食物 → 增加长度 → 加分 → 生成新食物
    ↓
撞墙/撞自己 → 游戏结束 → 显示结束提示
```

### 3. 用户操作
```
点击方向键 → 改变移动方向 → 防止反向移动
    ↓
点击暂停 → 停止游戏循环 → 显示暂停提示
    ↓
点击重新开始 → 重置游戏状态 → 重新开始
```

## 核心算法

### 移动算法
```typescript
// 1. 获取当前蛇头位置
head = snake[0]

// 2. 根据方向计算新头部位置
switch(direction) {
  case UP:    head.y -= 1
  case DOWN:  head.y += 1
  case LEFT:  head.x -= 1
  case RIGHT: head.x += 1
}

// 3. 检测碰撞
if (撞墙 || 撞自己) {
  游戏结束
  return
}

// 4. 添加新头部
snake.unshift(newHead)

// 5. 检测是否吃到食物
if (新头部位置 == 食物位置) {
  加分
  生成新食物
  // 不移除尾部，蛇变长
} else {
  移除尾部
  // 蛇长度不变
}
```

### 食物生成算法
```typescript
do {
  // 随机生成坐标
  food.x = random(0, gridWidth)
  food.y = random(0, gridHeight)
} while (食物位置在蛇身上)

// 确保食物不会生成在蛇身上
```

### 碰撞检测
```typescript
// 撞墙检测
if (head.x < 0 || head.x >= gridWidth ||
    head.y < 0 || head.y >= gridHeight) {
  return true // 撞墙
}

// 撞自己检测
for (segment of snake) {
  if (head.x == segment.x && head.y == segment.y) {
    return true // 撞到自己
  }
}

return false // 没有碰撞
```

## 性能指标

- **帧率**: 约 6.7 FPS (150ms/帧)
- **渲染方式**: 声明式UI + 状态驱动
- **更新机制**: 局部刷新（仅变化部分）
- **内存占用**: 极小（仅存储坐标数组）
